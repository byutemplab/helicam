Detailed Description

SysDesc.xml File

The SysDesc.xml file is located in directory C:/ProgramData/Heliotis/devDesc

SysDesc.xml contains informations about firmware, hardware and interface description. It is stored in a structure SysDesc.

tag 'System'

The tag 'System' is the root to describe a system that is composed off different objects:

<System id="c3cam_sl70" sensor="helioct3" fpga="c3cam_sl70" usb="c3cam"/>\n

the 'System' tag with id 'c3cam_sl70' refers to the objects:

  • the 'Sensor' tag with id='helioct3'
  • the 'FPGA' tag with id='c3cam_1012'
  • the 'USB' tag with id='c3cam_1010'

tag 'FPGA'

The tag 'FPGA' describe properties if the FPGA firmware:

<FPGA id="c3cam_sl70" regDesc="Reg_c3cam.xml" fpgaCfg="rtsd_xxxxxx_SL70.rbf"
verStr="...">
...
</FPGA>

in the the 'FPGA' tag with id='c3cam_sl70' the most important descriptions are:

  • fpgaCfg="rtsd_xxxxxx_SL70.rbf"
    describes which firware will be downloaded.
  • regDesc="Reg_c3cam.xml"
    describes which register description file is used.

Reg_c3cam.xml Files

Reg_c3cam.xml is a register description file. It is located in directory C:/ProgramData/Heliotis/devDesc

The register description describes the names and index of the registers and which bits/registers are mapped to a property. This is stored in a RegDesc structure.
Such a mapped property can be a bit, uint8, int16, enumerator etc. that can be accessed with HE_GetMap/HE_SetMap.

Here some lines of a register description:

<RegDesc>
<Reg num="0x03" id="AcqCtrl1" cmt="Acquisition control (volatile register)">
<Map id="CamMode" type="enum:0:3" mode="a">
<Enum num="0x00" id="raw IQ"/>
<Enum num="0x01" id="amplitude"/>
<Enum num="0x02" id="smooth amplitude"/>
<Enum num="0x03" id="intensity"/>
<Enum num="0x04" id="simple max"/>
<Enum num="0x07" id="min energy"/>
</Map>
<Map id="VolReady" type="bool:3" mode="rv" cmt="read only: indicates that a volume is ready for read out in the first mem"/>
<Map id="SensCfgBusy" type="bool:6" mode="rv" cmt="read only: indicates that configuration data is being downloaded and that camera should not be triggered"/>
</Reg>
<Reg num="0x06" id="HwCtrl" cmt="Hardware control">
<Map id="EnSynFOut" type="bool:0" cmt="0: Disables synchronization signal on OUT3 (rectangular waveform with demodulation frequency fd, see appendix A2) 1: Enables synchronization signal on OUT3"/>
<Map id="OutEnDrv" type="bool:1" cmt="0: Set OutEnDrv output to 0, 1: Set OutEnDrv output to 1 (Pin 5 on camera connector)"/>
<Map id="Out1" type="bool:2" cmt="0: Set general purpose output Out1 to 0, 1:Set general purpose output to 1"/>
<Map id="In1" type="bool:3" cmt="General purpose input In1 (read only)"/>
</Reg>
<Map id="SensTqp" type="uint16:12" cmt="tqp= fs/(8*fd)-30 with fs=70MHz(Sensorclock), fd (demodFreq)">
<Reg num="0x10" id="SensTqp0" bit="0" cmt="LSB of TQP register"/>
<Reg num="0x11" id="SensTqp1" bit="8" cmt="middle SB of TQP register"/>
<Reg num="0x12" id="SensTqp2" bit="16" cmt="MSB of TQP register (address reserved, not implemented)"/>
</Map>
</RegDesc>
