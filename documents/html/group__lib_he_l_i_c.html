int HE_GetSerials ( char *  serial)

This function counts all unused (closed) heliCam which are connected to the computer. For each unused heliCam is the serial number stored in the char pointer.

Returns
number of connectet and unused (closed) heliCams
Parameters
serialpointer to the serial number array. Each serialnumber have a maximum size of 16 characters (17 with '\0'). The maximum number of usb devices is 127. So the maximum pointer size which you need to allocate is 17*127. or simply: (number of connected devices)*(17bytes)
HE_FuncCB* HE_SetCallback ( HE_FuncCB *  cb)

HE_SetCallback() can set a user callback function of type HE_FuncCB that is called on special events. Such special events can be:

  • image buffers will be/have been changed.
  • a message should be displayed to the user
  • a register or buffer changes in the camera dialog box
  • etc. (for a list of all events look at HE_FuncCB) This allows the main program to reallocate image pointers and enter e.g. critical sections to avoid access of freed buffers.
Returns
this function returns the previous function pointer. This gives the possibility to forward not handled message to the default handling
See also
HE_FuncCB, HE_GetDefaultCallback()

int HE_Open ( HE_HANDLE *  heHdl,
int  mode,
void *  param 
)

Generic Open camera open function.

  • mode=0:'Open using the Device Description Files'
    param=string "sys=name fpga=name sensor=name usb=name eth=name motion=name regDesc=name verbose=name"
    Default usage is just to give the system name.
    These names are described in the SysDesc.xml file.
    so an HE_Open,(heHdl, 0, "sys=c3cam_1012") opens the system c3cam_1012.
  • mode=1: 'Open Stream'
    param=string "C:\\myStream.hbs"
  • mode=2: 'Open without downloading the fpga firmware'
    like mode=0 but without downloading the fpga firmware. To use this mode, it's necessary to initialize the camera with HE_Open(..) and mode=0 and then before you reopen with mode=2 you must call HE_Close()!! It's help you to reconnect the USB connectiviti to a running camera. A refresh wil be faster then mode=0 and the camera settings are retained.

mode=0: additional argument is a string with following definition:
"sys=name fpga=name sensor=name usb=name eth=name motion=name regDesc=name verbose=name"

See also
SysDesc.xml File

CamData* HE_GetCamData ( HE_HANDLE  heHdl,
uchar  idx,
uchar  addRef,
CamDataMeta *  meta 
)

Returns the pointer to the idx-th CamData. The idx has a value from 0 to SR_GetNumCamData()-1. If addRef==1 a reference is added to that buffer and it is not freed if the systems releases it. Details about the content of data can be extracted from the CamData structure. Usually index is 0 if it is the raw data from the camera. If the idx is out of boundary or the buffer is not allocated, a value of 0 is returned. If the meta is not 0, the structure is filled with the meta data.

CamData* HE_ProcessCamData ( HE_HANDLE  heHdl,
uchar  idx,
uint  mode,
void *  param 
)

Processes a CamData object that is located at position idx.
The CamData must be allocated before with HE_AllocCamData mode and params are the processing mode and parameters if needed.
Actual possibilities are (the source is always RAW data from the chip):
MODE 0 : Convert Raw Buffer to the desired output buffer (mainly same format, just resorting data) :

  • Volume DF_I16Q16 300x300xN (DP_ENCODER&DP_ORDER_3STRIP) -> Volume DF_I16Q16 300x300xN ()
  • Volume DF_A16 282x292xN (DP_ENCODER&DP_ORDER_3STRIP) -> Volume DF_A16 282x292xN ()
  • Volume DF_A8 282x292xN (DP_ENCODER&DP_ORDER_3STRIP) -> Volume DF_A8 282x292xN ()
  • Surface DF_A16Z16 280x292x2 (DP_ENCODER) -> Surface DF_A16Z16 281x293x2(DP_INTERP_CROSS)
  • Surface DF_Z16A16P16 (DP_ENCODER&DP_ORDER_3STRIP) -> Surface stack DF_Z16A16P16 280x292x(ExSimpMaxHwin*2+1)x3 ()

MODE 1: Offset-compensation of buffer idx with the (short*)data at param (short*)data must be a buffer with the size of the input buffer: 280x292(volume), 282x292(surface), 300x300(raw-IQ) do this 'in-place' -> output buffer == input buffer

  • Surface DF_A16Z16 (DP_ENCODER) -> Surface DF_A16Z16 (DP_ENCODER) (Param=Offset Correction NOT TESTED)

int HE_UpdateAcquTiming ( HE_HANDLE  heHdl,
AcquTiming *  at,
ushort  mode 
)

This Helperfunctions helps to set real timing values to the camera and vice versa. The Camera attributes SensTqp,SensNavM2 have not dimensions like seconds or Hertz. Further the FrameFrequence and Demodulation Frequency have further dependencies Therefore this function helps to set real values and/or returns real values that are set on the camera. the mode describes if the real values are set or/and get to/from the camera. It is an 'ored' value of AcquTiming
